将来は音ゲー・縁結びパズル・宇宙神社経営シムへ育てる。

# 世界観

* 八百万の神々 — あらゆるものに神が宿る、という日本の世界観 × 宇宙（星雲・銀河）
* ビジュアルの正本はクロちゃんデザインの2D版（docs/original-design.html）

## Phase 1: 創世瞑想（実装済みプロトタイプ = index.html）

* 暗闇の3D宇宙。星屑が漂う
* タッチ／クリックすると光の塵が集まり、星の種が生まれる
* 呼吸ガイド（4秒吸う→4秒吐く のリング表示）に合わせて画面に触れていると星が育つ
* 育ちきると「神様」が誕生（光の球体になり、空間を漂い始める）
* 生まれた神様の数がカウントされる（STARS BORN）
* BGMなし・環境音のみ（Web Audio APIで生成、ファイル不要）

## Phase 2: 奉納の舞（音ゲー・2026-07-13 設計確定）

* 呼び名は「奉納の舞」。縁を結んだ神様たち（八柱のうち出会った柱）が円座に並ぶ
* 太鼓の拍に合わせて、神様のまわりに光の環が近づく。**環が神様に重なる刹那に画面へふれると「祝詞」が届く**
  * 「ちょうど」が一点に定まる工夫（2026-07-13 本人フィードバック「ちょうどのタイミングがバラバラ」対応）:
    * 神様のまわりに**受けの環**がうっすら浮かび、動く環がそこに重なる瞬間が「ちょうど」
    * 環は**ちょうど2拍**かけて近づく（太鼓2発ぶん、と体でわかる）
    * ちょうどを過ぎた環は**止まらず**、神様の光に沈みながら消える（「止まった＝今」の誤解を生まない）
    * タップすると必ず**柏手**が鳴る（叩いた実感が返る。届けば鈴が重なる。外れても柏手だけは鳴る＝責めない）
  * タッチは画面のどこでもよい（狙い撃ちさせない。スマホで気持ちよく）。**こころえにも「画面のどこでもかまいません」と明記**（2026-07-13 本人フィードバック「どれをタップするのかわかりづらい」対応）
  * 環が近づいている神様は、ほのかに明るくなる（いまどの子に祝詞を捧げるのか、目でわかる）
  * 判定は「お見事」（±0.15秒）と「良き」（±0.38秒）の2つだけ。**外れても表示も減点もしない**（瞑想アプリなので責めない）（2026-07-13 いいともさん命名）
    * 窓は最初±0.3／±0.65にしていたが「外したのに届いて16/16になる」と本人指摘。甘すぎると数字の意味が消えるため締めた
  * 判定のことばは、**祝詞が届いた神様のそば**に出す（画面上部の固定位置だと、どの神様に届いたか分からないため）
  * 音作りの注意（2026-07-13）: 太鼓は基音だけだとスマホの小さいスピーカーで聞こえない（100Hz以下が出ない）。**倍音＋バチのアタックノイズを重ねて「ドン」を知覚させる**。お見事の鈴はシャラーンと長く、良きは短くつつましく（耳でも差がわかるように）。**奉納中は環境音（ドローン）を0.25→0.1に下げて打楽器を立たせる**
* 祝詞の数の意味: 奉納の回数と祝詞の累計は**御朱印帳に刻まれる**。将来は Phase 3（縁結び）で神様との縁の深さに使う構想（未確定。決めるときはこの正本を先に更新）
* 祝詞が届くたび、その神様の色の光紋が広がり、空間に彩りの靄が残る（成功するほど宇宙が彩られる）
* 音はすべてWeb Audio生成: 太鼓（拍の頭・低いドン）、鈴（祝詞が届いた音）、笙の和音（はじまりとおわりのひとふき）
* テンポ54BPM・8小節・祝詞16回で、ひと舞 約40秒（瞑想的な遅さを保つ）
* 入り口: スタート画面「奉納の舞をささげる」（1柱以上と縁があるとき表示）／お参り後の画面にもボタン
* はじめての時だけ「奉納のこころえ」（説明画面）を挟む
* 記録: 奉納した回数と届いた祝詞の累計を localStorage に保存（save.hounouCount / save.noritoTotal）
* 結果画面は「祝詞が、届きました。」＋届いた数。**スコア煽り・コンボ・ランクは入れない**

## Phase 3: 縁結び（むすび）— 詳細設計（2026-07-13 Fable 5設計・実装は未着手）

世界観の核は「むすひ（産霊）」— 結ぶことで新しいものが生まれる、日本神話の生成の霊力。
奉納で貯めた祝詞を使って神様同士の縁を結ぶ。**奉納（Phase 2）→ 祝詞 → 縁結び → 新しい神様 → また奉納**、の循環を作る。

### 遊びの流れ

1. 入り口: 御朱印帳に「縁をむすぶ」ボタン（**手持ちの祝詞8つ**で押せる。八にちなむ）
2. 一柱目・二柱目を選ぶ（出会った自然の八柱から。まだの柱は選べない）
3. **むすびの呼吸**: ひと呼吸（4秒吸う→4秒吐く）のあいだ画面に触れたまま（縁結びも瞑想の延長にする）
4. 二柱のあいだに光の橋がかかり、**むすびの神**が誕生（誕生カードは既存流用）
5. どの組み合わせでも必ず何かが生まれる（ハズレなし・責めない）。何が生まれるかは結ぶまでのお楽しみ

### データ・実装方針

* 保存: `save.noritoHeld`（手持ちの祝詞。奉納で＋、縁結びで−8）を新設。`save.noritoTotal` は累計の記録として残す（減らさない）
* むすびの記録: 合わせ神も `save.counts` に載せる（id例: `musubi-hi-kaze`。idはソート順で一意に）
* 同じ組み合わせは何度でも結べる（countsが増えるだけ。図鑑埋めの動機は初回）
* 御朱印帳に「むすびの章」を追加（28枠。未取得はシルエット）
* 祝詞の残高は御朱印帳と奉納の結果画面に表示

## 八十八柱の全体構成（2026-07-13 Fable 5設計・数の骨格）

「八十八」は末広がりの八が重なるめでたい数（米の字、四国八十八ヶ所）。構成は **8＋28＋24＋16＋12 ＝ 88**。

| 章 | 柱数 | 中身 | 出会い方 | 実装フェーズ |
|---|---|---|---|---|
| 自然の八柱 | 8 | 日・風・山・水・火・雷・月・星 | おみくじ（実装済み） | Phase 1 済 |
| むすびの二十八柱 | 28 | 二柱の縁から生まれる合わせ神 | 縁結び | Phase 3 |
| 節気の二十四柱 | 24 | 立春・雨水・啓蟄…二十四節気の神 | **その節気の期間だけ**おみくじに混ざる（リアル時間連動） | Phase 3.5 |
| 暮らしの十六柱 | 16 | 竈・道・市・言・糸・器・音・香など、人の営みの神 | 宇宙神社の営み（参拝者との縁） | Phase 4 |
| 守護神の十二柱 | 12 | 記紀の神々（実装済み） | 誕生日診断 | Phase 1.5 済 |
| **龍神様** | 数の外 | すべての流れを束ねる存在 | 八柱コンプの隠し | Phase 1 済 |

* 龍神様は88に**数えない**（八百万の流れそのものだから。物語上の格を守る）
* 節気の二十四柱は「いま、この2週間しか出会えない」= **毎日開く動機＋2週間ごとの発信ネタ**（年24回「新しい神様が降りてきた」と発信できる）
* 節気の神に出会い損ねても責めない: 来年また巡ってくる（めぐりの世界観。コンプは1年がかりの楽しみ）
* 暮らしの十六柱の名簿と、節気二十四柱の逸話は、各フェーズ着手時にこの正本へ先に書いてから実装する

## ガチャではなく「おみくじ」

* 課金ガチャは入れない（瞑想の世界観を壊すため・禁止事項）
* 星が生まれるとき、どの神様が宿るかはおみくじ: 吉60%→中吉25%→大吉13%→大大吉2%（=龍神様）
* 大大吉は八柱コンプリート前は大吉に格下げ（龍神様の物語上の格を守る）
* レア度が上がる要素は課金ではなく「丁寧な瞑想」（時間帯・呼吸モードで出る神様が変わる）

## 守護神診断（2026-07-08 実装・正本。資料は docs/guardian-source.md）

三層構造の真ん中を担う要素。**誕生日で決まる、あなただけの一柱**。

* 資料の計算式で、生年月日から 1〜9・11・22・33 の12種の守護神（記紀の神々）を導く
  * 手順: 生年の各桁を足す → 月を足す → 日を足す → 2桁なら1桁化。ただし途中で 11・22・33 になったらマスターナンバーとしてそのまま
  * 例: 1997年12月22日 → (1+9+9+7=26)+12+22=60 → 6+0=6 → 「6: 伊邪那美命」
* 守護神12柱: 1須佐之男命 2月読命 3天照大御神 4神産巣日神 5大黒天 6伊邪那美命 7思金神 8伊邪那岐命・伊邪那美命 9恵比須天 11弁財天 22高御産巣日神 33天之御中主神
* **おみくじには混ぜない**。誕生日を入れたときだけ出会える特別枠
* 御朱印帳の最上部に「あなたの守護神」として常に鎮座。瞑想空間でもひときわ大きく漂う“お守り”
* 診断結果は localStorage に保存（save.guardian = 番号）。誕生日は保存しない（計算後は番号だけ持つ）
* ビジュアルは当面 光球＋名前。記紀の神々の3D人物モデルは将来のAI生成フェーズで

### 三層構造（キャラクターの全体像）
* 自然の八柱（日の神…）= 毎日出会う光の神々（おみくじ）
* 守護神12柱（天照大御神…）= 誕生日で決まる、あなただけの一柱

## お参りフロー（2026-07-07 いいともさん発案・正本）

「8秒押しっぱなしで即誕生」は忙しすぎたため、瞑想と誕生を分離した。

1. モード選択 → 「呼吸のこころえ」（説明画面。チェックで次回から非表示＝上級者向け）
2. **呼吸セッション**: リングに合わせて3呼吸（画面は触らない。瞑想に集中）
   * 吐くときのガイド:「はぁー」と小さく声に出して、否定的なエネルギーが息と一緒に出ていくイメージで（いいともさん直伝）
   * 裏付け: 吐く息を長くする・声に出す（ため息・ハミング）は迷走神経を刺激し副交感神経が優位になる
3. 3呼吸完了 → **星迎え**: 画面に8秒触れたままにすると一柱誕生（ここだけ触る＝儀式）
4. 誕生カード → 「もうひと巡り」or「今日はここまで」
5. 「今日はここまで」→ 締め画面「今日の分は、整いました」＋連続お参り日数

* 1回のお参り ≒ 1〜1.5分。3巡で約3分の瞑想になる（瞑想の推奨時間2〜3分に合致）
* 回数制限はしない（瞑想したい人を止めない）。かわりに**その日最初の一柱はおみくじ2回引きの良い方**（毎日開く動機）
* 連続お参り日数（streak）をlocalStorageに記録

# 技術方針

* Three.js（CDN読み込み・ES Modules）。HTML1枚で動く。ビルド不要
* スマホのタッチ操作を最優先（クリック＝タッチ）
* 音はWeb Audio APIで生成（音声ファイルを持たない）
* 収集データ（生まれた神様の記録）は localStorage に保存
* 神様のビジュアル: 当面はコード生成（光球＋粒子、龍神様は粒子のリボン）。
  Phase 3の図鑑あたりでAI 3D生成（Meshy / Tripo等で2D版の絵からGLB化）を検討
